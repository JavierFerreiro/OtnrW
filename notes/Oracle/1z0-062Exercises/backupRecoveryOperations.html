Backup and recovery operations

Officials

1-Consistent Server-Managed Backup. Perform a full offline backup using RMAN.

  1. Set the environment variables, ORACLE_SID, ORACLE_HOME and PATH and launch rman and connect using O.S authentication.
    export ORACLE_SID=orcl
    export ORACLE_HOME=/u01/app/oracle/product/12.1.0.2/dbhome_01
    export PATH=$PATH:$ORACLE_HOME/bin
    rman target /
  2. Shut down the DB and mount it and perform a full backup and study the feedback.
    shutdown immediate;
    backup database;
  3. Open the database.
    alter database open;
    exit;

2-Perform Server-Managed Open Backups (hot backup)
  1. Connect to rman
    rman target /
  2. Back up a tablespace using defaults.
    backup tablespace sysaux;
  3. Back up your archive logsfiles, againg using defaults.
    backup archivelog all;

3-Use the DRA to diagnose and repair a problem. First, we will cause a problem and then use DRA to fix it.
  1. Connect to DB wit rman and confirm that exist a backup of the sysaux tablespace.
    rman target /
    list backup of tablespace sysaux;
  2. Shutdown the instance and in the O.S. delete the datafile for SYSAUX tablespace.
    shutdown immediate;
    exit;
    rm /u01/app/oracle/oradata/orcl/sysaux01.dbf
  3. Connect with sqlplus and try to startup the instance (stop in mount mode). Then connect with rman
    sqlplus / as sysdba
    startup
    exit;
    rman target /
  4. Diagnose the problem, generate an advice and repair the failure
    list failure;
    advise failure;
    repair failure;
