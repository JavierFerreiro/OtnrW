1-Simulate a situation: You have been asked to take management of a database that you have not seen before, and for wich
the documentation is woefully inadequate. Write a series of queries that will begin to document the system. Following are some
of the views that help. Try to resolve this questions:

  1. On what O.S. is the db running?
  2. Where is the controlfile? Is it multiplexed?
  3. How many online log file groups are there?
  4. How many numbers are in each group, and what are they called?
  5. How big are they?
  6. What tablespaces exist in the database?
  7. What datafiles are assigned to each tablespace?
  8. What are they called and how big are they?
  1. cat /etc/system-release
  2. select status,name from v$controlfile;
  3. select group#,members from v$log;
  4. select group#,status,member from v$logfile;
  5. select group#,bytes/1024/1024 as MB from v$log;
  6. select tablespace_name from dba_tablespaces;
  7. select tbs.tablespace_name, df.file_name from dba_tablespaces tbs,dba_data_files df where tbs.tablespace_name=df.tablespace_name;
  9. select file_name,bytes/1024/1024 as MB from dba_data_files;
