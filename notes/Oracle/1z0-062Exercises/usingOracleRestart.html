Using Oracle restart

Officials

1-Register a DB with Oracle Restart (must: complete previous exercises 20-1)

  1. From O.S. with root user check HAS is configured and running:
    export PATH=$PATH:/u01/app/12.1.0/grid/bin
    crsctl config has crsctl check has
  2. Demonstrate that the GI process that implements Oracle Restart is itself protected against failure:
    ps -ef|grep init.ohasd
    root 1148 1 0 08:48 ? 00:00:00 /bin/sh /etc/init.d/init.ohasd run >/dev/null 2>&1 </dev/null
    kill -9 1148
    ps -ef|grep init.ohasd
  3. Use srvctl to register your DB with Oracle Restart
    srvctl add database -d orcl -oraclehome $ORACLE_HOME
  4. Confirm that the database has benn succesfully registered and check its status
    srvctl config database -d orcl
    srvctl status database -d orcl
  5. Force GI to perform a reality check by issuing a start command and then check the status again:
    srvctl start database -db orcl
    srvctl status database -db orcl
  6. While the db is running demonstrate the restart capability:
    ps -ef| grep smon
    oracle 6312 1 0 13:02 ? 00:00:00 ora_smon_orcl
    kill -9 6312
    srvctl status database -db orcl
    srvctl status database -db orcl
